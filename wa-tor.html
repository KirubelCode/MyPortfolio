Learn More About the Wa-Tor Simulation

Overview

The Wa-Tor Simulation models a predator-prey ecosystem using a toroidal grid. It features multi-threaded behavior through Go's goroutines and synchronisation tools. Fish and sharks move, breed, and interact based on set rules, making it a real-time demonstration of concurrent programming principles.

Key Features

  • Concurrency: Efficient grid updates with goroutines and sync tools.
  • Configurable: Set custom values for grid size, creature counts, breeding, and starvation.
  • Toroidal World: Movement wraps around the grid edges.
  • Benchmarking: Execution time logs help evaluate concurrency effects.

Installation & Usage

Prerequisites: Install the latest version of Go.

Clone the repository:

git clone https://github.com/KirubelCode/WaTor-Project.git
cd WaTor-Project

To run with defaults:

go run .

To customise parameters:

go run main.go 100 100 3 3 4 100 8

Parameters:

  • NumFish: Number of fish
  • NumSharks: Number of sharks
  • FishBreed: Breeding interval for fish
  • SharkBreed: Breeding interval for sharks
  • Starve: Time before sharks die without food
  • GridSize: NxN grid
  • Threads: Number of threads (goroutines)

Technologies Used

  • Go (Golang): Core application and concurrency logic
  • sync.WaitGroup: Handles synchronisation of goroutines
